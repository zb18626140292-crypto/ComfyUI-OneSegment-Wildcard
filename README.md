# comfyui-文本自定义节点

一个包含两个实用文本节点的 ComfyUI 自定义节点插件：

- `多TXT只抽一段（通配符）`
- `多文本框拼接（Join String Multi）`

## 安装

1. 解压 `comfyui-文本自定义节点.zip`。
2. 将整个 `comfyui-文本自定义节点` 文件夹放到 ComfyUI 的 `custom_nodes` 目录。
3. 重启 ComfyUI，并在浏览器中按 `Ctrl+F5` 强制刷新前端。
4. 在节点搜索中输入 `多TXT只抽一段` 或 `多文本框拼接`。

本机示例安装位置：

```text
E:\ComfyUI_portable_TE_v260619\ComfyUI\custom_nodes\comfyui-文本自定义节点
```

从旧版 `ComfyUI-OneSegment-Wildcard` 升级时，请先退出 ComfyUI，再用新目录替换旧目录；节点内部 class ID 保持不变，已有工作流可以继续使用。

## 文件列表界面

- 节点顶部只有一个“上传或拖入 TXT 文件”区域，不显示路径文本框。
- 可一次选择或拖入多个 TXT，数量不限；文件会复制到 `wildcards/uploads`。
- 所有本节点实例共享读取 `wildcards/uploads`：创建节点、切换工作流、上传、删除、窗口重新获得焦点时会立即同步，并每 3 秒检查一次外部文件变化。
- 新增或删除的上传文件会同步到所有节点；各节点已有文件的开关状态和拖拽顺序仍独立保存。
- 每个 TXT 显示为独立条目：
  - 左侧开关：打开时参与随机抽取，关闭时保留文件但不参与抽取。
  - 中间文件名：鼠标停留可查看节点内部路径。
  - 右侧 `×`：立即从列表移除；对于上传文件，同时删除 `uploads` 中的副本。
- 按住文件行中部或 `≡` 拖动，可以改变 TXT 顺序；顺序会随工作流保存。
- 光标会按区域变化：上传、开关和删除为 `pointer`，文件拖拽区域为 `grab`，拖动过程中为 `grabbing`；离开节点后恢复默认光标。
- 文件条目及开关状态会随 ComfyUI 工作流一起保存。
- 左侧开关带约 180ms 的滑动和颜色渐变动画，动画不会改变节点尺寸。
- 上传区和文件行使用 Canvas/LiteGraph 原生绘制，会跟随节点缩放；增删文件后节点高度会自动重新计算，避免 DOM 控件越界和底部大片空白。
- 绘制宽度被强制限制在节点逻辑宽度内，以兼容会错误传入画布显示宽度的前端版本；浅色主题下上传文字和文件名使用深色显示。
- 上传、删除、刷新或载入工作流时保留当前节点宽度，并把高度紧凑到实际内容高度。
- 删除采用右侧按钮与节点级命中双重处理，并先从列表移除；即使上传副本已经不存在，也不会阻止删除条目。
- 删除期间会把路径加入全局待删除集合，所有节点和定时同步都会暂时忽略该文件，避免同步请求把它重新加入；磁盘确实删除失败时会显示具体原因并恢复条目。
- 节点高度以最后一个实际显示控件的位置为准，每次增删或载入后自动紧凑，清除底部遗留空白，同时保留当前宽度。
- 右下角保留约 26 像素缩放空间，原生缩放热区扩大到 28 像素，并显示三条斜线抓手；鼠标进入时显示缩放光标。

## 抽取设置

- `分段方式`：
  - `每个非空行是一段`：默认选项，适合普通 Easy Use 通配符 TXT。
  - `自动`：TXT 中有空行则按空行分段，否则每个非空行是一段。
  - `空行分段`：适合一个提示词由多行组成、段落之间留空行的 TXT。
- `抽取方式`：
  - `每个文件等概率`：先随机文件，再随机该文件的一段；不会让行数多的文件占优势。
  - `所有片段等概率`：合并所有片段后统一随机。
- `seed`：默认值为 `1`，`生成后控制` 默认为 `randomize`；之后可像采样节点一样自由选择固定、递增、递减或随机，选择会随工作流保存。
- 种子控件使用与 Easy Use“随机种”一致的范围，并提供 `🎲 Manual Random Seed` 按钮；点击后立即生成一个固定随机种并运行一次。
- 旧版节点第一次载入时，会修复缺少“抽取方式”造成的控件序列化错位，并把旧默认的 `seed 0/1 + fixed` 一次性迁移成 `seed 1 + randomize`；之后不会覆盖手动设置。

输出：

- `文本`：抽中的唯一一段，可接 CLIP Text Encode 或其他文本节点。
- `来源文件`：显示内容来自哪个 TXT。
- `段落序号`：该段在来源文件中的序号，从 1 开始。
- `顺序组合文本`：从每个已启用 TXT 各抽一段，并按节点中的文件顺序用换行拼接。

上传、开关和删除文件都不需要重启 ComfyUI。

## 多文本框拼接（Join String Multi）

这个节点参考 [KJNodes 的 Join String Multi](https://github.com/kijai/ComfyUI-KJNodes)，但把动态字符串入口直接做成了可编辑的多行文本框，不依赖 KJNodes 插件。

- `inputcount`：文本框数量，范围 2–1000；修改后会立即增减 `string_N` 文本框。
- `delimiter`：拼接分隔符，默认是一个空格。
- `return_list`：关闭时输出拼接后的单个字符串；开启时输出字符串列表。
- `string_1`、`string_2`……：可直接输入多行文本；除第一个文本框外，空文本框会在拼接时跳过。
- 文本框内容和数量会随工作流保存；临时减小数量后，在本次页面会话中重新增加还能恢复刚才隐藏的内容。
